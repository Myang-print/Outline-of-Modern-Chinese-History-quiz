# `docs/`

Project documentation and generated review materials.

## Structure

- `USAGE.md`: Local startup, preview, build, and verification commands.
- `QUESTION_BANK_PIPELINE.md`: Raw data, annotation, generated output, and maintenance workflow.
- `question-bank/`: Generated question-bank reports, review drafts, and study notes.

## Update Rules

- Keep durable project documentation in Markdown.
- Generated question-bank documents should be regenerated from scripts rather than hand-edited.
- If a generated document exposes an issue, fix the parser, source data, or annotation source instead.
